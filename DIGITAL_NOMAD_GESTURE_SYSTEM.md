## 🤚 手势识别核心创新

### 为什么选择手势？

**传统技术局限**:
- **脑机接口(EEG)**: 高成本、侵入性强、伦理争议、远离生活
- **现有穿戴设备**: 重测量而非理解、缺乏具身性、控制性过强

**手势优势**:
- **具身性**: 手势是身体的自然语言，揭示真实情感状态
- **预知性**: 在意识察觉前捕捉微反应变化
- **实用性**: 日常生活、非侵入、深层人性化
- **隐私友好**: 轻量级、非控制性交互

### 核心设计哲学
> "手势读取身体而非控制身体" - 让技术成为理解伙伴而非管理者

## 🔄 一日节奏手势映射

### 早晨 (6:00-9:00) - 苏醒准备阶段
**手势模式**:
- 伸展手腕/缓慢咖啡杯 = 身体苏醒，准备工作
- 轻柔手指活动 = 从休息向工作状态过渡

**技术特征**:
- EMG: 低张力、渐进激活
- GSR: 稳定状态、温和上升

### 中午 (12:00-14:00) - 认知疲劳阶段
**手势模式**:
- 敲手指敲击/快速键盘输入 = 高认知负荷
- 肩部紧张/姿势僵硬 = 压力累积

**技术特征**:
- EMG: 高频激活、持续性紧张
- GSR: 不规律、高峰响应

### 下午 (15:00-17:00) - 注意力分散阶段
**手势模式**:
- 姿势微调/手部躁动 = 需要休息
- 频繁触摸设备/眼神游离 = 专注力下降

**技术特征**:
- EMG: 不规则激活、疲劳模式
- GSR: 低频波动、恢复缓慢

### 晚上 (19:00-22:00) - 恢复放松阶段
**手势模式**:
- 放松握持/轻触杯子 = 释放压力
- 温和手势/呼吸配合 = 进入休息状态

**技术特征**:
- EMG: 低张力、放松状态
- GSR: 稳定恢复、基线回归

#### 检测阶段 (Detection)

```python
def detect_embodied_signals():
    # EMG信号处理 (1000Hz采样)
    emg_features = extract_emg_features(
        rms=calculate_rms(window),
        mdf=calculate_mdf(window),
        zc=calculate_zero_crossings(window),
        wl=calculate_waveform_length(window)
    )

    # GSR信号处理 (100Hz采样)
    gsr_features = extract_gsr_features(
        scl=calculate_skin_conductance_level(window),
        scr=calculate_skin_conductance_responses(window),
        peaks=detect_peaks(window),
        recovery_time=calculate_recovery_time(window)
    )

    return fuse_features(emg_features, gsr_features)
```

#### 干预阶段 (Intervention)
```python
def gentle_intervention(state, confidence):
    if state == "fatigue_detected" and confidence > 0.7:
        # 温和反馈 - 光脉冲/声音/微振动
        return deliver_rhythm_guidance("rest_prompt")
    elif state == "stress_detected" and confidence > 0.8:
        return deliver_rhythm_guidance("breathing_guidance")
    return None  # 无干预
```

### 数据处理要求

#### 帧率优化
- **实时性**: >60Hz处理帧率，确保流畅响应
- **平滑处理**: 滑动窗口平滑，避免误触发
- **延迟控制**: 总延迟<200ms，用户感知不到

#### 数据来源
- **主要数据**: 现实用户采集 (15人4周研究)
- **辅助数据**: 公开EMG/GSR数据集进行模型训练
- **验证数据**: WESAD、NinaPro等标准数据集

## 🎨 用户界面设计

### 原则: 感知优先，控制最小

#### macOS应用 (主监测端)
